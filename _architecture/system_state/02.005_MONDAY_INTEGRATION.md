# Monday.com Integration Addendum
## Replaces All Notion References in the Idea Intake Workflow

**Document Version**: 001
**Effective**: 2026-03-21

---

## What Changed

Monday.com replaces Notion as the planning and tracking tool across all Silva Wood Holdings
businesses. This affects the Idea Intake Workflow in these specific ways:

### 1. METHODOLOGY_ROADMAP.md Now Mirrors Monday.com Boards

Previously: snapshot of Notion Kairos Actions database
Now: snapshot of Monday.com boards, organized by METHOD area

Each Monday.com board maps to a METHOD area. The roadmap file includes:
- Board registry (ID, name, method area, item count, URL)
- Per-board item tables with Monday.com item IDs
- Dependency chains per method area

### 2. Seed Files Cross-Reference Monday.com Items

When a seed corresponds to an existing Monday.com item, the seed file includes:
```
**Monday.com Item**: 18404773839 / 11558478888
```
Format: `[Board ID] / [Item ID]` or full URL.

When a seed is a brand new idea with no Monday.com item yet, leave blank.
After the seed is approved and moves to Active, create the Monday.com item
and update the cross-reference in both directions:
- Seed file gets the Monday.com item ID
- Monday.com item gets a Docs link to the workdoc or spec

### 3. Status Tracking Lives in Monday.com, Not BACKLOG_INDEX.md

The BACKLOG_INDEX.md remains the **local quick-reference** for Claude Code,
but Monday.com is the authoritative source for:
- Item status (Ideate, Backlog, Active, On Hold, Blocked, Done, Canceled)
- Priority
- Time estimates
- Assigned business / project
- ECDs (estimated completion dates)

BACKLOG_INDEX.md tracks the **development pipeline status** (which phase of
the 20/80 workflow each idea is in), which Monday.com doesn't know about:
- Seeds (not yet analyzed)
- In Analysis (Claude Code expanding)
- Ready (spec'd, awaiting build)
- Active (currently being built)
- Complete (archived with outcomes)

These are complementary, not redundant. Monday.com tracks the *what* and *when*.
BACKLOG_INDEX tracks the *development phase*.

### 4. CLAUDE.md Addition — Updated Reference

Replace the Notion reference in root CLAUDE.md with:

```markdown
## Planning & Tracking

- **Monday.com** is the planning tool for all businesses (silvawoodholdings.monday.com)
- **_architecture/METHODOLOGY_ROADMAP.md** is the local snapshot for Claude Code context
- The backlog lives in `_architecture/_backlog/` as markdown files
- BACKLOG_INDEX.md is the single source of truth for development pipeline status
- When a seed becomes an active build, ensure the Monday.com item has a Docs link
  back to the local spec, and the local seed has the Monday.com item ID
```

### 5. Sync Process

Monday.com → Local snapshot sync can happen two ways:

**Manual** (current): Silva or Claude Code reads Monday.com boards and updates
METHODOLOGY_ROADMAP.md during planning sessions.

**Automated** (future): A Python script in `00_shared/tools/` that:
1. Reads registered board IDs from METHODOLOGY_ROADMAP.md
2. Fetches current items and statuses via Monday.com API
3. Regenerates the per-board item tables
4. Writes the updated METHODOLOGY_ROADMAP.md
5. Commits to git

This automation is itself a backlog seed when the time is right.

## Monday.com Board Naming Convention

To keep things navigable as boards multiply:

```
[Method Area] - Actions
```

Examples:
- Market Screener - Actions (METHOD.01)
- Entry Framework - Actions (METHOD.08)
- Backtesting - Actions (METHOD.10)
- Social Media - Actions (METHOD.13)

For non-Method businesses:
```
[System Name] - Actions
```

Examples:
- Growth Hub - Actions
- SaaS Platform - Actions
- Consulting Ops - Actions

## Monday.com Column Standard

All ACTION boards should use this consistent column set:

| Column | Type | Purpose |
|--------|------|---------|
| Name | name | Action title |
| Status | status | Ideate / Backlog / Active / On Hold / Blocked / Done / Canceled |
| Estimate | status | M15 / M30 / H1 / H2 / H4 / Day / Multi-Day |
| Notes | long_text | Description and implementation notes |
| Docs | link | Link to workdoc, spec, or external reference |
| ECD | date | Estimated completion date |
| Business | text | Which business entity |
| Project | text | Which project/method area |

This matches what's already on the Market Screener board (18404773839).

## The Idea (What + Why)

Build a new PyQt6 application that serves as the single morning workflow tool. Silva sits
down, opens the app, and all pre-computed data from Supabase is ready to review — screener
table, charts with levels and H4 supply/demand zones, and bucket run status. This replaces
the current pre-market report (alpha) with a more capable tool that integrates the bucket
pipeline (Seed 004), adds H4 supply/demand zone visualization, and scales to the full
50-ticker universe.

## Trigger (What Made You Think of This)

Seed 004 built the bucket runners that pre-compute all screener data into Supabase. The
data is there — but there's no consumer application that reads it all back and presents it
in a single cohesive workflow. The current pre_market_report.py is a Streamlit-style tab
inside the existing PyQt6 app. The new tool needs to be standalone, purpose-built for the
morning review workflow: see the screener table, cycle through charts, pick 4 tickers,
and go.

Additionally, the H4 supply/demand zones (`00_shared/calculations/h4_supply_demand/`)
are fully built and tested but NOT integrated into the confluence scoring or the chart
visualization in the bucket pipeline. This tool will be the first to display them as
chart overlays alongside the confluence zones.

## Desired Outcome

1. **Bucket Monitor Panel** — Shows last run time and status for Weekly/Nightly/Morning
   buckets. Red/green indicators per bucket. Shows which tickers failed if any.

2. **Manual Trigger Buttons** — Run Weekly, Nightly, or Morning bucket on demand from
   the UI. Progress bar and log output visible.

3. **Screener Table** — Reads pre-computed data from Supabase. Columns include:
   ticker, composite direction, D1/H1/M15 structure, price, D1 ATR, zones count,
   primary setup direction/POC/target/R:R, RVOL, GAP. Sortable and filterable.
   Additional columns to be added later.

4. **Chart Viewer** — Click a ticker in the screener table, see its H1 chart with:
   - HVN POC levels (from hvn_pocs table)
   - Camarilla pivots (from bar_data table)
   - H4 Supply/Demand zones (computed and overlaid — orange supply, green demand)
   - Market structure strong/weak levels
   - Pre-market high/low/POC (from pm_* columns in bar_data)
   - Zone boundaries (from zones table)

5. **Ticker Cycling** — Arrow keys or next/prev buttons to cycle through tickers
   in screener order. Chart updates instantly (data is pre-loaded from Supabase).

6. **Pre-market prep time: <10 minutes** — Open app, scan table, cycle charts, pick 4.

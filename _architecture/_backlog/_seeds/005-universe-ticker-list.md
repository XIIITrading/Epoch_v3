## The Idea (What + Why)

Define the base ticker universe that feeds the screener pipeline (Seed 004). This is the list of tickers that Bucket A (weekly) processes and Bucket B (nightly) computes data for. The universe scanner (Seed 004 Phase 3) monitors this list during market hours.

Starting point: Silva's existing daily click-through view tickers. The list will be stored in Supabase (`screener_universe` table) and updated weekly or as needed.

## Trigger (What Made You Think of This)

Seed 004 (Screener Pipeline Build) needs a defined ticker universe to process. The universe selection criteria were originally part of Seed 004 but separated into their own seed because:
1. The ticker list definition is a trading methodology decision, not an infrastructure decision
2. Silva has an existing daily click-through view that serves as the starting point
3. The list may evolve independently of the pipeline infrastructure

## System Area

- `01_application/` — Universe stored in Supabase, consumed by bucket runners
- Supabase table: `screener_universe` (new table defined in Seed 004)

## Desired Outcome

1. A defined list of tickers stored in `screener_universe` table
2. Each ticker has: symbol, sector (optional), status (active/inactive), epoch anchor date
3. Epoch anchor dates auto-detected using existing "High Volume Day within 6 Months" (`find_max_volume_anchor()`)
4. Silva provides the initial list; future updates can be manual or criteria-based
5. Seed 001 (Base Ticker Universe) is subsumed by this seed — the methodology criteria live here

## Priority Signal

**MEDIUM** — Needed before Seed 004 Phase 3 (50-ticker scanner) can be tested with real data. Not blocking Phase 1 or Phase 2 (which can use manually entered tickers).

## Constraints

- **Depends on**: Seed 004 Phase 1 (creates the `screener_universe` table and bucket runner that reads it)
- **Starting input**: Silva's daily click-through view tickers (to be provided)
- **Epoch anchors**: Use existing `find_max_volume_anchor()` — no new algorithm
- **Size target**: ~50 tickers (exact number driven by data and Silva's judgment)

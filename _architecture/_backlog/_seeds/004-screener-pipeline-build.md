## The Idea (What + Why)

Restructure the screener pipeline from a single monolithic pre-market calculation into four
time-based execution buckets (Weekly, Nightly, Morning Session, Real-Time). Pre-compute all
higher-timeframe data into Supabase on schedule so the morning session becomes a fast query
rather than a full calculation run. This unlocks scaling the real-time Entry Qualifier from
4 tickers to the full 50-ticker universe during market hours.

## Trigger (What Made You Think of This)

The pre-market workflow currently takes 30-45 minutes of active work because everything —
monthly/weekly OHLC, ATR, Camarilla, volume profiles, market structure, zone scoring — is
computed in a single pass when the app launches. Most of this data doesn't change intraday
and could be pre-computed overnight or weekly. The Seed 003 audit confirmed that 24 of 45
total calculations already exist and just need scheduling into the right bucket. The
remaining 10 new calculations (pre-market volume profile, W1/M1 structure, universe
selection, epoch anchor auto-detection) can be built against the same infrastructure.

Additionally, the Entry Qualifier is limited to 4 tickers because it fetches and computes
everything live. With higher-timeframe data pre-loaded in Supabase, the live scanner only
needs to poll M1 bars and compute 7 lightweight indicators per ticker — making 50-ticker
monitoring feasible within Polygon API rate limits (~12.5 seconds per 60-second cycle).

## System Area

- `01_application/` — Pipeline runner refactored into bucket-based scheduler
- `00_shared/indicators/` — Shared indicator library (volume profile for pre-market, EMA)
- `02_dow_ai/entry_qualifier/` — Scaled from 4 to 50 tickers, reads pre-computed data from Supabase
- `00_shared/data/supabase/` — New tables or columns for pre-market and weekly data
- Supabase tables: `bar_data`, `hvn_pocs`, `market_structure`, `zones`, `setups` + potential new tables

## Desired Outcome

1. **Bucket A (Weekly)** runs Saturday/Sunday — computes monthly/weekly OHLC, Camarilla,
   W1/M1 market structure, epoch anchor auto-detection, and 50-ticker universe selection.
   Results stored in Supabase.

2. **Bucket B (Nightly)** runs after 20:00 ET — computes D1 OHLC, all ATR values, D1
   Camarilla, options OI, PDV volume profile, HVN POCs, D1/H4/H1/M15 market structure,
   zone confluence scoring, and setup generation. Results stored in Supabase.

3. **Bucket C (Morning)** runs 07:00-08:00 ET — computes pre-market high/low, pre-market
   volume profile (PMPOC/PMVAH/PMVAL) using bars from 16:00 ET prior day to 07:30 ET,
   current price snapshot. Queries Bucket B data from Supabase.

4. **Bucket D (Real-Time)** runs 09:30-16:00 ET — lightweight M1 polling for up to 50
   tickers. Computes candle range %, volume delta, volume ROC, SMA config, and M5/M15/H1
   fractal structure (raw labels + levels, not composite scores). All higher-timeframe
   context pre-loaded from Supabase.

5. **Pre-market prep time drops from 30-45 minutes to <10 minutes** (query + review, not
   compute).

6. **Entry Qualifier scales to full universe** during live session.

## Priority Signal

**HIGH** — This is the build that follows directly from Seed 003 (audit). The audit
confirmed feasibility and mapped every calculation to its bucket. This seed turns that
map into running code.

## Constraints

- **Depends on**: Seed 003 (Screener Bucket Audit) must be complete — provides the
  authoritative data contract and bucket assignments
- **Polygon API budget**: Massive tier. 50-ticker live polling at 0.25s rate limit =
  12.5 seconds per 60-second cycle — confirmed feasible
- **No scoring systems**: LONG/SHORT composite scores (0-7) are deprecated per SWH-6.
  Real-time signals use raw fractal structure labels (BULL/BEAR/NEUTRAL) + strong/weak
  levels from `shared/indicators/structure/`
- **EMA 9/21 is Bucket D only**: Not pre-computed nightly. Computed live as part of the
  existing SMA config indicator
- **Pre-market window**: 16:00 ET prior day to 07:30 ET current day (for PM volume profile
  and PM high/low)
- **Must not break**: Existing backtest pipeline (03_backtest reads zones/setups),
  existing Entry Qualifier functionality for current 4-ticker use case
- **Existing overnight calculation**: Currently uses 20:00 UTC to 12:00 UTC — may need
  alignment to the 16:00-07:30 ET pre-market definition

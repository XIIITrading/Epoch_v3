# Data Journal — Improvements Log

Running log of actionable findings from the data journal analysis suite.
Each entry captures what the data showed, the proposed change, and implementation status.

## Structure

- Each file is dated: `YYYY-MM-DD_improvements.md`
- Findings accumulate over time as the dataset grows
- Prior findings are revisited when sample sizes increase
- Status: PROPOSED → TESTING → IMPLEMENTED → REJECTED

## How to Use

1. Run `/data-journal` to generate fresh analysis
2. Compare results against prior improvement entries
3. Add new findings or update confidence on existing ones
4. Track which improvements have been implemented in the screener/pipeline
